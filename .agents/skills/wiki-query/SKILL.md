# wiki-query

wiki를 탐색하여 질문에 답한다.

## 사전 준비

1. `wiki/WIKI_SCHEMA.md`를 읽어 wiki 구조를 파악한다.
2. `wiki/index.md`를 읽어 현재 어떤 페이지들이 있는지 파악한다.

## 질문 유형과 탐색 전략

### 포스트 찾기

"이 주제 다룬 포스트 있어?", "CRDT 관련 글 뭐 썼지?"

1. `wiki/index.md`에서 관련 source와 topic을 찾는다.
2. 해당 페이지들을 읽고 관련 포스트 목록을 제시한다.
3. 필요하면 원본 포스트(`raw/{folder}/index.mdx`)를 읽어 상세 내용을 확인한다.

### 관점/생각 질문

"나는 AI에 대해 어떻게 생각해?", "내 개발 철학이 뭐야?"

1. 관련 topic 페이지를 찾아 읽는다.
2. topic의 "개요"와 "인사이트" 섹션에서 종합된 관점을 답변한다.
3. 근거가 되는 source 페이지를 함께 인용한다.

### 성장/변화 질문

"이 주제에 대한 생각이 어떻게 바뀌었어?", "최근 1년간 어떻게 성장했어?"

1. 관련 topic의 "변화와 흐름" 섹션을 읽는다.
2. 회고 kind의 source들을 시간순으로 읽는다.
3. 변화의 궤적을 정리하여 답한다.

### 인사이트 요청

"앞으로 뭘 더 공부하면 좋을까?", "내 블로그에서 빠진 주제가 뭐야?"

1. 전체 topic 목록을 훑어 현재 다루는 영역을 파악한다.
2. topic의 "인사이트" 섹션에서 기존 인사이트를 수집한다.
3. 빈 영역, 미완의 탐구, 발전 가능한 방향을 제안한다.

### 페르소나 탐구

"나는 어떤 개발자야?", "내 블로그로 나를 소개해봐"

1. 전체 topic과 주요 source를 읽는다.
2. kind 분포, 관심 주제, 반복 패턴, 고유한 관점을 종합한다.
3. "이런 사람이다"를 구조적으로 답한다.

## 답변 원칙

- 답변에는 근거가 되는 wiki 페이지를 인용한다.
- wiki에 없는 내용은 추측하지 않고, "wiki에 이 주제에 대한 정보가 없습니다"라고 답한다.
- 필요하면 원본 포스트를 직접 읽어 wiki보다 더 상세한 답변을 제공할 수 있다.

## 답변 저장

유용한 답변이 나왔을 때, 사용자에게 wiki에 저장할지 제안한다.

- 분석/비교 결과 → `wiki/topics/{topic-name}.md`로 저장하거나 기존 topic 갱신
- 새로운 인사이트 → 관련 topic의 "인사이트" 섹션에 추가

저장하면 `wiki/index.md`와 `wiki/log.md`도 갱신한다.

log 형식:

```markdown
## [YYYY-MM-DD] query | 질문 요약

답변을 {페이지경로}에 저장. (또는: 저장 없이 답변만 제공)
```
