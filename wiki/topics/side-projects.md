## 개요

사이드 프로젝트는 성장의 실험장이자 즐거움의 원천. 디자인 시스템, 블로그, 에디터, 피그마 플러그인 등 다양한 프로젝트를 통해 새로운 기술을 탐구하고 역량을 넓힌다.

## 관련 포스트

- [2024-developer-retrospect-2](../sources/2024-developer-retrospect-2.md) — Terra Design System, 블로그 재도전, Ticketbell, Readme Editor
- [2025-developer-first-half-retrospect](../sources/2025-developer-first-half-retrospect.md) — 블로그 자동화, Ticketbell 종료
- [2025-retrospect-2](../sources/2025-retrospect-2.md) — Variables for i18n, Scrapper, Readme Editor 재시작

## 변화와 흐름

**2024**: 디자인 시스템에 대한 깊은 탐구(KRDS → Terra). 블로그 4년 만에 재도전 성공. Ticketbell에서 "덜어내기" 원칙 적용.

**2025**: AI 덕분에 사이드 프로젝트 생산성 향상. 피그마 플러그인이라는 새로운 영역 진출. React Native에도 도전. 흥미가 식으면 중단했다가 AI 덕분에 재시작하는 패턴.

## 인사이트

- 흥미 기반으로 시작하고 흥미가 식으면 중단하는 패턴이 있다. AI의 등장으로 zero-to-one 비용이 낮아져 중단됐던 프로젝트가 다시 살아나는 경우가 생김.
  근거: [2025-retrospect-2](../sources/2025-retrospect-2.md)

- 일상의 불편함에서 출발하여 도구를 만드는 것이 자연스러운 패턴. 이것이 "에디터 만드는 게 즐겁다"는 정체성과 연결됨.
  근거: [2024-developer-retrospect-3](../sources/2024-developer-retrospect-3.md), [2024-developer-retrospect-2](../sources/2024-developer-retrospect-2.md)
