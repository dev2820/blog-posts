HTML만으로는 충분하지 않고 Json ld가 필요한 이유

```
HTML
= 사람에게 보여줄 문서의 구조를 마크업하는 언어

JSON-LD / Linked Data
= 컴퓨터가 의미를 이해할 수 있게 정보를 구조화해서 제공하는 데이터
```

## JSON-LD란

- JavaScript Object Notation for Linked Data

JSON에 특정 규약을 먹여 Linked Data로 만든 것이다. 웹 브라우저 혹은 서버에서 정보를 주고 받을 때 유용하게 쓸 수 있다.

## Semantic Web

## Linked Data & RDF

Linked Data[^4]란 웹의 창시자 Tim Berners Lee가 만든 **웹 상에 데이터를 올릴 때 권장되는 규칙 4가지**를 지칭한다.

1. 무언가 이름 붙일 때는 URI를 써라 (Use URIs as names for things)
2. HTTP URI를 써서 사람들이 이 것들을 찾아볼 수 있게 하라 (Use HTTP URIs so that people can look up those names)
3. 그 주소에 접근하면 유용한 정보를 제공하라 (When someone looks up a URI, provide useful information)
4. 더 많은 정보를 발견할 수 있도록 다른 URI 링크를 포함해라 (Include links to other URIs. so that they can discover more things)

요약하면
가령 "Terra가 PostA를 작성했다."라는 문장은 "PostA - 작성자 - Terra Yang" 관계를 맺는다.

## RDF란?

Resource Description Framework[^5]의 약자로 웹 상에서 정보를 주고 받기 위해 고안된 표준화된 모델이다.
정보를 주어 - 동사 - 목적어로 표현하는데 예를 들면 다음과 같다.

Alice - live - Seoul(엘리스는 서울에 산다)

## Web에서 RDF의 의미

필자의 글에 reference가 달리는 것처럼, Web은 문서와 문서들이 링크로 연결된다. 기계가 웹사이트를 읽고나면 웹사이트가 포함한 링크를 통해 다른 사이트로 연결된 것은 알 수 있지만, 이 연결이 무엇을 의미하는지는 알 수 없다. 어쩌면 악의적인 게시자에 의해 스팸 주소가 연결된걸지도 모른다.

## 왜 필요한가?

일반적인 HTML은 다음과 같이 생겼다.

```html
<h1>JSON-LD란 무엇인가?</h1>
<p>작성자: Terra Yang</p>
<p>이 글은 JSON-LD와 Linked Data를 설명합니다.</p>
```

우리는 h1에 적힌 내용이 구글에서 페이지의 제목으로 나온다는 것을 알지만, 사실 HTML은 마크다운 언어일 뿐, h1이 해당 페이지의 제목이라는 규약을 보장하지는 않는다.
따라서 기계가 p 태그를 보고는 'Author가 Terra Yang이구나!'하고 스스로 이해하지는 못한다.

p 태그에 의미 부여를 해준다면 어떨까? 가령
(중략)

반면 JSON으로 표현한다면

```json
{
  page: [
    {
      type: 'title',
      value: 'JSON-LD란 무엇인가?
    },
    {
      type: 'author',
      value: 'Terra Yang'
    },
    {
      type: 'summary',
      value: '이 글은 JSON-LD와 Linked Data를 설명합니다.'
    }
  ]
}
```

위와 같이 의미를 부여할 수 있다.
